## 一、SendSmsVerifyCode（发送短信验证码）

### 接口说明
- 由于运营商近期加强对短信签名的管控，自定义签名面临下发失败问题，推荐使用号码认证控制台赠送的短信签名和模板。**系统赠送签名必须搭配系统赠送模板使用**
- 短信认证服务仅收取短信发送费用（按运营商回执状态计费，提交成功但回执失败不计费），**核验服务免费**

### 调试入口
OpenAPI Explorer：https://api.aliyun.com/api/Dypnsapi/2017-05-25/SendSmsVerifyCode
（可直接在线运行，免去手动计算签名，运行成功后能自动生成各语言SDK代码示例）

### 授权信息（RAM权限策略用）
`dypns:SendSmsVerifyCode`，访问级别 create，资源类型：全部资源

### 请求参数

| 名称 | 类型 | 必填 | 描述 | 示例值 |
|---|---|---|---|---|
| SchemeName | string | 否 | 方案名称，不填则为"默认方案"，最多20字符 | 测试方案 |
| CountryCode | string | 否 | 号码国家编码，默认86，目前仅支持国内号码 | 86 |
| PhoneNumber | string | 是 | 短信接收方手机号 | 130****0000 |
| SignName | string | 是 | 签名名称，**暂不支持自定义签名，请使用系统赠送的签名**（控制台"赠送签名配置"页面选择） | 速通互联验证码 |
| TemplateCode | string | 是 | 短信模板CODE，选择赠送签名时必须搭配赠送模板 | 100001 |
| TemplateParam | string | 是 | 短信模板参数，验证码位置两种传法：`{"code":"##code##","min":"5"}`（由系统生成，可被接口校验）或 `{"code":"123456","min":"5"}`（自己指定值，**阿里云接口无法校验**） | {"code":"##code##","min":"5"} |
| SmsUpExtendCode | string | 否 | 上行短信扩展码，系统自动生成，不支持自行传入 | 1213123 |
| OutId | string | 否 | 外部流水号 | 外部流水号（透传） |
| CodeLength | integer | 否 | 验证码长度，支持4~8位，默认4位 | 4 |
| **ValidTime** | integer | 否 | **验证码有效时长，单位秒，默认300秒**（⚠️见上方"待核实事项"） | 300 |
| DuplicatePolicy | integer | 否 | 同场景同号码重复发送时旧验证码处理规则：1=覆盖（默认，旧验证码失效）2=保留（多个验证码都在有效期内可校验通过） | 1 |
| Interval | integer | 否 | 发送频控间隔，单位秒，默认60秒 | 60 |
| CodeType | integer | 否 | 验证码类型（仅TemplateParam传占位符时生效）：1=纯数字（默认）2=纯大写字母 3=纯小写字母 4=大小写混合 5=数字+大写字母 6=数字+小写字母 7=数字+大小写字母 | 1 |
| ReturnVerifyCode | boolean | 否 | 是否在返回结果里带上验证码明文 | true |
| AutoRetry | integer | 否 | 是否自动替换签名重试：1=开启（默认，运营商返回明确失败时自动换方式重试）0=不开启 | 1 |

### 返回参数

| 名称 | 类型 | 描述 | 示例值 |
|---|---|---|---|
| AccessDeniedDetail | string | 访问被拒绝详情 | 无 |
| Message | string | 状态码描述 | 成功 |
| RequestId | string | 请求ID | CC3BB6D2-... |
| Model.VerifyCode | string | 验证码（若ReturnVerifyCode=true才有） | 4232 |
| Model.RequestId | string | 请求ID | a3671ccf-... |
| Model.OutId | string | 外部流水号 | 1231231313 |
| Model.BizId | string | 业务ID | 112231421412414124123^4 |
| Code | string | 请求状态码，`OK`代表成功 | OK |
| Success | boolean | 请求是否成功 | true |

### 错误码

| HTTP状态码 | 错误码 | 错误信息 | 描述 |
|---|---|---|---|
| 400 | MOBILE_NUMBER_ILLEGAL | The mobile number is illegal. | 手机号码格式错误 |

## 二、CheckSmsVerifyCode（核验验证码）

### 调试入口
OpenAPI Explorer：https://api.aliyun.com/api/Dypnsapi/2017-05-25/CheckSmsVerifyCode

### 授权信息
`dypns:CheckSmsVerifyCode`，资源类型：全部资源

### 请求参数

| 名称 | 类型 | 必填 | 描述 | 示例值 |
|---|---|---|---|---|
| SchemeName | string | 否 | 方案名称，**若发送接口传了非空方案名，这里必须一致** | 测试方案 |
| CountryCode | string | 否 | 号码国家编码，默认86 | 86 |
| PhoneNumber | string | 是 | 手机号 | 186****0000 |
| OutId | string | 否 | 外部流水号 | 12123231 |
| VerifyCode | string | 是 | 用户输入的验证码。若发送时TemplateParam用的是`##code##`占位符（系统生成），可以正常校验；若发送时直接传了具体数值（自己指定），**阿里云接口无法校验** | 1231 |
| CaseAuthPolicy | integer | 否 | 大小写核验策略：1=不区分（默认）2=区分 | 1 |

### 返回参数

| 名称 | 类型 | 描述 | 示例值 |
|---|---|---|---|
| AccessDeniedDetail | string | 访问被拒绝详情 | 无 |
| Message | string | 状态码描述 | 成功 |
| Model.OutId | string | 外部流水号 | 1212312 |
| **Model.VerifyResult** | string | **核验结果：`PASS`=成功，`UNKNOWN`=失败**（这是本项目`sms-verify.js`判断依据） | PASS |
| Code | string | 接口状态码，`OK`=请求成功（⚠️不代表核验成功，核验结果只看`Model.VerifyResult`） | OK |
| Success | boolean | 接口调用是否成功（⚠️同上，不代表核验成功） | true |
| RequestId | string | 请求ID | CF8854E5-... |

**重要提醒（官方原文强调两次）**：接口请求成功（`Code: OK` / `Success: true`）**不等于**验证码核验通过，必须看 `Model.VerifyResult === 'PASS'` 才算真正通过。

## 三、单用户QPS限制

官方文档提到（在 GetSmsAuthTokens 接口页面注明，推测同样适用于本产品线）：单接口 QPS 限制 5000次/秒，超过会被限流。对个人/小微项目几乎不可能触及，仅作记录。
