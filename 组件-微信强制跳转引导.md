# 组件：微信内打开强制跳转引导（wechat-mask）

> 定稿于 2026-07-01，来自 gold-mate 项目 V1.8.14 ~ V1.8.30 的迭代打磨。
> 本文档是**独立、自包含**的组件说明——不需要读取本项目的对话记录或其他文档，
> 直接照着"三、完整代码"那节复制粘贴，改两处文案，就能在新项目里用。

---

## 一、这个组件解决什么问题，什么时候该用它

**适用场景**：你的PWA/H5产品需要用户"添加到手机桌面"才能获得完整体验（或者必须
跳出微信才能完成某个关键动作，比如支付），但用户经常是从抖音/小红书/微信公众号
这类渠道、在**微信内置浏览器**里打开链接的——而微信内置浏览器**无法**执行"添加
到主屏幕"，很多第三方支付方式在微信内置浏览器里也跑不通。

**这个组件做的事**：检测到当前是在微信里打开，立即用一个**无法关闭**的全屏遮罩
挡住整个页面，引导用户点击右上角"···"→"在浏览器打开"，跳转到系统浏览器后才能
看到真正的页面内容。

**这不是一个"更好的体验提示"，是一个"没有它，产品核心功能就用不了"的强制关卡**
——如果你的产品在微信内置浏览器里能正常使用（不需要强制跳出去），不要用这个
组件，改用更轻量的、可关闭的提示即可，硬拦截会平白流失一批用户。

## 二、这次迭代踩过的坑，直接决定了下面代码为什么长这样

在照抄代码之前，务必看这几条——都是真实踩过的坑，不是理论上的顾虑：

1. **不要做"假图标徽章"去模拟真实的系统按钮样子。** 最早的版本做了两个圆形
   图标去模拟微信自己的"···"按钮和分享面板里的"浏览器"图标，结果导致"假图标
   位置对不对"和"箭头指向准不准"两个坐标都要精确，出错概率是叠加的。最终版本
   放弃了假图标，图标只是嵌在文字句子里的一个小符号，箭头单纯负责"往这个方向
   看"，不需要精确戳中任何东西。

2. **箭头永远不可能精确指向真实的系统按钮。** "···"是微信自己原生工具栏上的
   元素，不在你的页面DOM里，你拿不到它的真实像素坐标，只能凭经验估算，而且
   会因设备型号、微信版本、状态栏高度、甚至系统自带的翻译工具栏弹出与否而产生
   偏差（这几种情况在实测中都真实遇到过）。**箭头的设计目标是"传达大致方向"，
   不是"精确指向"**，不要花时间去做像素级精确定位，那是徒劳的。

3. **不要用"猜哪里是空的"来保证文字可读，除非确认这片区域内容不会变。** 文字
   区块下面透出的是产品自己的页面内容（比如金价数字），这些内容长度、位置会
   随数据变化，直接把文字叠在半透明遮罩上，实测被页面自己的大号数字"顶穿"、
   叠在一起很乱。解决方法有两种：要么给文字加一块实色背景衬底（更稳妥，但视觉
   上更"重"），要么像本项目最终选择的那样，把容器宽度留够、让长文字不换行，
   减少纵向占用的空间来避开碰撞——**两种方法都可以，取决于你更在意视觉简洁还是
   绝对稳妥**，本项目最终选了后者。

4. **箭头的SVG坐标必须验算绝对位置，不能只调曲线形状。** 调整箭头弧度、长度的
   时候，容易只关注"这段曲线好不好看"，却忘了重新计算箭头顶端（也就是最关键的
   指向点）在整个屏幕里的**绝对像素位置**有没有跑到视口边界外面被裁切掉。每次
   改动箭头坐标后，建议心算一下：`外层容器的top值 + SVG自身的top偏移值`，
   确保结果是正数、且留有安全余量（本项目最终版本留了20px）。

5. **多个动画共用同一个CSS class很危险。** 如果项目里有别的地方也用了同一个
   浮动/跳动动画class（比如这个项目iOS引导里另一个箭头图标也用了`.pwa-bounce`），
   不要为了单独调整这个组件的动画速度/幅度而直接修改共用的class定义，那样会
   连带影响到其他不相关的地方。给每个需要独立调参的地方各自建一个专属class。

## 三、完整代码（复制粘贴即可用）

### 3.1 前置依赖

需要在 `<head>` 里加载中文字体（如果新项目字体方案不同，把下面代码里所有
`font-family:'Noto Sans SC',sans-serif` 换成你自己的字体即可）：

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400&display=swap" rel="stylesheet">
```

**关于字重（300/400/700）的选择，是个可以自己决定的事，不是必须修的问题**：
下面代码里`font-weight:700`那行文字，如果字体链接没有加载700这个字重，浏览器
会用"伪加粗"（拉伸400字重模拟）来渲染——这确实不是真正的粗体字形，但本项目
实测对比过两种效果后，**最终选择保留伪加粗**，因为真实粗体字形在实际测试中
视觉效果反而不如预期。如果你想要真正的粗体，把上面链接里的`wght@300;400`
改成`wght@300;400;700`即可；如果没改，就是继续用伪加粗，这本身也是一个完全
可以接受的选择，不是bug，实际效果如何建议自己测一下再决定。

### 3.2 CSS（放进 `<style>` 里）

```css
@keyframes wechatArrowBounce {
  0%, 100% { transform: rotate(60deg) translateY(0); }
  50% { transform: rotate(60deg) translateY(-10px); }
}
.wechat-arrow-bounce { animation: wechatArrowBounce 0.5s ease-in-out infinite; }
```

### 3.3 HTML（放进 `<body>` 里，位置不限，建议放在接近 `</body>` 的地方）

```html
<div id="wechat-mask" style="display:none;position:fixed;top:0;left:0;width:100%;height:100%;background:rgba(0,0,0,0.82);z-index:9999;touch-action:none;">

  <!-- 内容区域：可以调整 top/right/width 来适配不同产品的页面布局，
       核心原则见上面"二、踩过的坑"第3条——如果页面内容会变化，容器
       要留够宽度让长文字不换行，避免纵向碰撞 -->
  <div style="position:absolute;top:80px;right:70px;width:280px;">

    <!-- 虚线弧形箭头：传达"往右上角看"的大致方向，不追求精确指向 -->
    <svg class="wechat-arrow-bounce" style="position:absolute;top:-60px;right:-40px;width:80px;height:82px;" viewBox="0 0 80 82">
      <path d="M69,78 C73,52 68,23 39,5" stroke="white" stroke-width="2.5" fill="none" stroke-dasharray="6,5" stroke-linecap="round"/>
      <polygon points="34,2 44,4 37,11" fill="white"/>
    </svg>

    <!-- 两行文字：小字铺垫 + 大字加粗指令，图标嵌在句子里，不做假图标徽章 -->
    <div style="text-align:center;margin-bottom:28px;">
      <div style="font-family:'Noto Sans SC',sans-serif;font-size:13px;color:rgba(255,255,255,0.85);margin-bottom:8px;">
        点击右上角「···」
      </div>
      <div style="font-family:'Noto Sans SC',sans-serif;font-size:18px;font-weight:700;color:white;line-height:1.4;">
        选择在浏览器 🌐 中打开
      </div>
    </div>

  </div>
</div>
```

### 3.4 JS（放进 `<script>` 里，页面加载时执行一次即可）

```javascript
// 检测微信浏览器，显示引导遮罩
(function() {
  const ua = navigator.userAgent.toLowerCase();
  const isWeChat = ua.indexOf('micromessenger') !== -1;
  if (isWeChat) {
    document.getElementById('wechat-mask').style.display = 'block';
  }
})();
```

## 四、新项目里怎么调用——三步走

1. **复制**上面3.1~3.4四段代码，分别粘贴到新项目的对应位置（字体link、style、
   body里的HTML、script）
2. **改文案**：如果新产品要引导的动作不是"添加到桌面"，把HTML里两行文字的
   具体措辞改成符合你产品场景的话术，保持"小字铺垫+大字加粗指令"这个结构
   不变即可
3. **按需调整定位数值**：`top:80px;right:70px;width:280px`这几个数值是照着
   本项目页面顶部信息栏的具体高度调出来的，新项目的页面布局不一样，大概率
   需要重新微调这几个值——**改完之后务必按"二、3、4条"里说的方法重新验算**，
   不要凭感觉直接照搬数字

**不需要额外调用任何函数或初始化步骤**——这四段代码本身就是完整闭环，页面
加载后JS会自动判断是否在微信里、自动决定要不要显示这个遮罩，全程不需要手动
触发。

## 五、这个组件不包含什么（如果你的新项目也需要，得另外做）

- **不包含**"是否已经添加到桌面"的判断逻辑——这个遮罩只负责"检测到在微信里
  就拦截"，跟用户装没装桌面图标无关，每次在微信里打开都会拦
- **不包含**跳出微信之后、在真正的系统浏览器里该展示什么引导内容——那是另一
  套逻辑（iOS/Android分别的"添加到主屏幕"步骤说明），本项目里这部分逻辑挂在
  一个叫`install-guide`的独立弹窗上，跟这个组件没有直接关联，如果新项目也
  需要，建议另起一份文档单独整理，不要混在一起
